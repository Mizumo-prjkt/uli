# ULI Composite YAML Standard (v1.0)

This document defines the schema for the composite profiles generated by the ULI Compositor and consumed by the ULI Installer.

## Structure

The file is a single YAML document containing a list of `components`.

```yaml
version: "1.0"
components:
  - id: string          # Unique identifier (e.g., "nginx-webserver")
    name: string        # Display name in the UI
    description: string # Detailed description
    category: string    # UI grouping (e.g., "Web Servers")
    
    # Distribution-specific package mappings
    packages:
      arch: [string]
      debian: [string]
      alpine: [string]
    
    # Post-install autostart routines per init system
    autostart:
      systemd: [string]
      openrc: [string]
      sysvinit: [string]
      
    # Metadata for the compositor
    metadata:
      last_sync: timestamp
      source_repo: string
```

## Field Definitions

| Field | Type | Description |
| :--- | :--- | :--- |
| `id` | String | A kebab-case unique identifier for the component. |
| `packages` | Object | Map of distribution IDs to arrays of package names. |
| `autostart` | Object | Map of init-system IDs to arrays of shell commands. |

## Example Entry

```yaml
components:
  - id: "docker-engine"
    name: "Docker Container Runtime"
    description: "The industry-standard container engine."
    category: "Virtualization"
    packages:
      arch: ["docker", "docker-compose"]
      debian: ["docker.io", "docker-compose"]
      alpine: ["docker", "docker-compose"]
    autostart:
      systemd: ["systemctl enable --now docker"]
      openrc: ["rc-update add docker default", "rc-service docker start"]
```
